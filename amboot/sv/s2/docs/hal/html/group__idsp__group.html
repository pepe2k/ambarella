Detailed Description

API calls to query the frequency of the idsp clock.

The idsp clock frequency cannot be set directly. It is changed by AMBHAL when the operating mode is changed. See also Operating Mode.

Get the status of the previous requested idsp pll frequency change.

Note:
A new idsp pll frequency change may be requested after this function returns AMB_HAL_SUCCESS.
Parameters:
[in]amb_hal_base_addressVirtual address where ambhal is loaded by OS.
Return values:
AMB_HAL_SUCCESSthe pll has locked to the new frequency.
AMB_HAL_FAILthe pll lock has failed to lock in a reasonable amount of time. something is wrong.
AMB_HAL_RETRYthe pll has not locked yet. try again.
